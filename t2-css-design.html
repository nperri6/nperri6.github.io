CSS

An Interactive Exploration

August 10th, 2015

This week I had the chance to start building my website! While I have worked to build websites before using content management systems, this was one of my first attempts from scratch. I learned a lot about CSS this week, mostly from trial, error, and a whole lot of googling. Here is some of the stuff that I have learned!

Margins, Borders, and Padding

Marigns, borders, and padding are all unique, and all quite important for positioning objects. Margins describe the space around the object itself in relation to the objects around it. Padding is slightly different. It still describes space around an object, but this space is considered part of that object. In a way it expans the area of the object (including the clickable area for a link). Border describes the space inbetween padding and margin where you can create borders if you would like.

Classes vs. Ids!

One of the extremely helpful features that HTML/CSS offers is classes/ids. Whenever you have more than one element that you want to style similarly with CSS, you can apply the class to its tag. For example, class="mainheader". Then, you style that in css using the .mainheader {}. Ids are very similar but generally reserved for one specific div. If you have specific styling you only need to apply to one section of your website, IDs are the way to go. Ex. id="specificsidebar" CSS: #specificsidebar {}

Relative, Absolute, and Fixed Positioning

Positioning was one of the tougher things for me to wrap my head around this week. While it isn't too hard to theoretically understand what things should do, it doesn't always work as expected (but hopefully as my CSS knowledge grows they will). Fixed positioning means that a section will not move. It will be anchored to the screen in its position. Absolute positioning means that it will have a specific position on the screen, though will not be fixed ( if you scroll you should be able to scroll past it). Relative positioning means that it will be positioned relative to its parent element and what else is on the page. If you position a relative element inside an absolute element, it will position itself inside that absolute element relative to the borders of that element.

Display: Inline vs Inline Blocks

Inline basically means that elements will line themselves up on a line. However, if those elements do not have any inherit traits that tell them to occupy space, the will end up lining up on top of each other. In these cases you would need to use inline blocks, which tells those elements to occupy blocks of space.

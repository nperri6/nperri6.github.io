Sept 13, 2015

In the programming world there is a lot of discussion about which language is the "best" to use. In reality there is no one language that is good for every task. If there was, wouldn't we stop creating new languages? Both Javascript and Ruby are incredibly useful, but for different reasons. They have some overlapping characteristics, but differ in style and functionality.

One big difference between the two is the syntax that you need to use to write in either language. Ruby was created with the developer in mind. Due to this, it is fairly easy to learn the syntax. Ruby is very forgiving with its requirements on punctuation. With ruby you can use () with functions, or you could choose not to. You can space things out, or you can choose not to. The best practices for ruby actually use as little punctuation as possible. This is a pretty typical function definition in Ruby 

 
	    def hello
	    	puts "Hello"
	    end

Notice how little punctuation there is? Meaning there is none! Now lets take a look at what you would have to write with JS to do the same thing 


	    var hello = function(){
	     console.log("Hello");
	     }

As you can see Javascript requires a lot more syntax to write the code out. This isn't necessarily a bad thing, but it does take some getting used to! (especially if you started with ruby) There are tons of other differences between the two languages, but I feel they c an all be boiled down to the idea that Ruby was made for developers, while JS was made to modify web pages. The JS language has grown and evolved over time to incorporate more user friendly services, but its fundamentally created to serve a purpose of modifying HTML. Ruby was created with the developer in mind so it tends to be easier on the eyes!

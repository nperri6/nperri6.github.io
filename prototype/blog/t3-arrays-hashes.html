Horray for Array!

Arrays give us our first opportunity for storing more than 1 thing together as an object. Arrays are great for storing multiple things that are of the same class, such as strings, fixnums, etc., although each item in an array does not have to be of the same class!

Here is what an array looks like in ruby: 


	    array = []  # if you want to create an empty array
	    array =Array.new  # another way to do the same thing
	    array = [1,2,3,4,5]  # define an array with things in it
	    array << 6      # push new things into your array

Another great way of storing multiple items in a single ruby object is through a hash. You can think of a hash as the dictionary of ruby objects. Just as a dictionary has a word and a definition, a hash contains a key and a value. Each key is unique but the values they can contain may overlap.(think synonyms) The great thing about hashes is that they aren't limited to holding strings or fixnums. You can actually store arrays or other hashes in the values of a hash! This can get kind of complex, but it adds a lot of power to what you can do with your ruby hashes!

Here is what a hash might look like in ruby: 


 	    hash = {} #empty hash
 	    hash = Hash.new #anther empty hash
 	    hash = {time: 5, day: monday} # populate a hash
 	    hash[day] = 23   #give a hash key a value
When to use a hash vs an array

Hashes and arrays both hold information, but have different uses. Whenever you want to simply store a list of objects then you should use an array. However, if you are needing to store dictionaries of items (for instance each item has some information about it) then you would want to use a hash. Hashes have a lot more utility for storing information about poeple. For instance if you were making an address book you would want to use a hash. There you could set the key to the persons name and in the value you could store all their information (you could even use an array or hash with street name, city, state, phone # etc.!)
